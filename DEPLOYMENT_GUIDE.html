CompanionHub Deployment Guide

This guide will help you deploy the CompanionHub application to GitHub Pages.

Note: This application consists of both a frontend (React) and backend (Node.js). GitHub Pages can only host static files, so you'll need to deploy the backend separately or use a service like Heroku, Vercel, or Render for the backend.

Step 1: Create a GitHub Repository

  1. Go to GitHub and log in to your account
  2. Click the "+" icon in the top right corner and select "New repository"
  3. Name your repository (e.g., "companionhub")
  4. Choose to make it public or private
  5. Do not initialize with a README, .gitignore, or license
  6. Click "Create repository"

Step 2: Update Git Remote URL

Replace "YOUR_USERNAME" with your actual GitHub username in the following command:

git remote set-url origin https://github.com/YOUR_USERNAME/companionhub.git

Step 3: Push Code to GitHub

git push -u origin master

Step 4: Deploy Frontend to GitHub Pages

  1. Navigate to your repository on GitHub
  2. Go to Settings > Pages
  3. Under "Source", select "GitHub Actions"
  4. GitHub will automatically suggest a workflow for deploying a React app
  5. Click "Configure" and then "Start commit"
  6. Commit the workflow file

Step 5: Configure Environment Variables

For the frontend to work properly, you'll need to set environment variables:

  1. Go to your repository Settings
  2. Click "Secrets and variables" > "Actions"
  3. Add repository secrets for any API keys or environment variables your app needs

Alternative Deployment Options

For Backend Deployment:

  • Heroku: Free tier available, easy deployment
  • Render: Free tier with automatic SSL
  • Vercel: Great for Node.js APIs
  • Railway: Modern deployment platform

For Frontend Deployment:

  • GitHub Pages: Free, integrated with GitHub
  • Vercel: Excellent for React apps, free tier
  • Netlify: Great features, free tier
  • Firebase Hosting: Fast, reliable hosting
Important: Make sure to update all API endpoints in your frontend code to point to your deployed backend URL instead of localhost.

Troubleshooting

  • If you get permission errors, make sure you're using the correct GitHub credentials
  • Check that your repository name matches exactly
  • Ensure Git is properly installed and configured
  • Verify your internet connection

Need Help?

If you encounter any issues during deployment, check the GitHub documentation or reach out to the GitHub community for support.
